# Модуль 7. Локальные модели для разработки

## Цель модуля
Научить участников выбирать, запускать и оценивать локальные open-weights LLM в инженерном контуре.

## Длительность
3 часа (45 мин теория, 60 мин демонстрации, 75 мин практика)

## Темы
- Инженерная экономика локальных моделей.
- Ландшафт open-weights LLM _(as of Q2 2026)_.
- Ollama, LM Studio, vLLM.
- Железо, квантование и VRAM-арифметика.
- Локальный code review.
- Data governance, security, TCO.

## Демонстрация
1. Установка модели через Ollama/LM Studio.
2. Проверка latency, качества и требований к железу.
3. Сравнение с облачной моделью на одинаковом промпте.

## Практика 1
**Задача**: локально развернуть модель и выполнить code review одного модуля.

**Критерии приемки**:
- Review содержит минимум 5 замечаний с приоритетами.
- Минимум 2 замечания подтверждены вручную.
- Отмечены ложноположительные срабатывания.

## Архитектурные акценты для senior
- Data governance: какие данные можно отправлять локальной модели.
- Security: защита секретов в локальном контуре.
- Стоимость владения: железо, поддержка, обновление моделей.
- Когда локальный стек лучше облака, и наоборот.

## Простые термины
- **Локальная модель**: LLM, запущенная на собственном железе, а не через облачный API.
- **Open-weights**: модель с публично доступными весами.
- **Квантование**: снижение разрядности весов для экономии памяти.
- **VRAM**: видеопамять GPU.

## Типичные ошибки
- Ожидание, что локальная модель всегда дешевле и точнее.
- Игнорирование лицензий open-weights моделей.
- Отсутствие verifier-loop при локальном code review.

## Артефакт модуля
Рабочий локальный code-review workflow для одного модуля проекта.
